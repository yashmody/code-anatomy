# `docs-site/` — DEPT® Anatomy of Code v2 documentation

This is the Docusaurus 3 source for the v2 docs site. **Phase 0 scaffold only.**
Phase 5a runs `npm install`, writes the section content, and wires the build
into the Apache vhost.

## What is here

```
docs-site/
├── package.json                # Docusaurus 3.5.x + theme-mermaid deps (pinned)
├── docusaurus.config.js        # title, navbar, ochre theme, mermaid enabled
├── sidebars.js                 # six top-level categories
├── src/css/custom.css          # DEPT® brand tokens — ochre, Syne, DM Sans, JetBrains Mono
├── static/img/README.md        # placeholder; logo + favicon land here in Phase 5a
└── docs/
    ├── intro.md
    ├── frontend/               # 1 · Front-end
    ├── content-architecture/   # 2 · Content architecture
    ├── database/               # 3 · Database
    ├── deployment/             # 4 · Deployment
    ├── quiz-management/        # 5 · Quiz management
    └── system-architecture/    # 6 · System architecture
```

Each section folder has:
- `_category_.json` — sidebar label, position, link target.
- `intro.md` — the section landing page (Scan Box + planned pages).

## Two commands to run it (Phase 5a)

```bash
cd docs-site
npm install        # one-time
npm run start      # dev server on http://localhost:3000/docs/
```

Production build:

```bash
npm run build      # outputs to docs-site/build/
npm run serve      # serves the build for smoke testing
```

`build/` is what Apache serves at `/docs/` (see the Phase 5a checklist in
`docs/architecture/v2/08-docs-plan.md`).

## Where the build artefacts go

- `docs-site/build/` is generated by `npm run build`. It is gitignored.
- `infra/deploy.sh` (Phase 5a) rsyncs `docs-site/build/` to the deploy
  target and the Apache vhost adds `Alias /docs/ /path/to/build/`.

## Brand

- Accent: `#FF4900` (DEPT® ochre).
- Display: Syne. Body: DM Sans. Mono: JetBrains Mono.
- Logo of record: `https://www.deptagency.com/wp-content/uploads/2025/10/logo-dept.svg`.

These are mirrored in `src/css/custom.css` and must stay in sympathy with the
main app (`frontend/styles/monolith.css`).

## Authoring policy

See `docs/architecture/v2/08-docs-plan.md` (in the repo, not on this site)
for the full authoring policy. The short form:

- Indian English spelling.
- LAYER pattern: Scan Box (3-5 bullets, ~30-second read) at the top, then
  prose, then diagrams and callouts woven through.
- Hybrid diagrams: ASCII for static topology, Mermaid for flows.
- Four callout types only: *Why This Matters*, *Agency Tip*, *Common
  Pitfall*, *Before / After*. They map to Docusaurus admonitions.
